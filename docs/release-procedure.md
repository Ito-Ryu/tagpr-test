# モジュールリリース手順

## 概要

このドキュメントでは、開発者がモジュールを修正してから、リリースされるまでの一連の手順とルールを説明する。

## 手順

### 1. 開発・マージ
機能追加やバグ修正の Pull Request を作成し、`main` ブランチにマージする。

### 2. リリース用 PR の作成
`main` ブランチに変更が push（マージ）されると、GitHub Actions が実行され、`tagpr` が自動的にリリース用の Pull Request を作成または更新される。

このPRには次期バージョンのタグ付けとリリースノートの準備が含まれる。

### 3. リリース
リリース用 Pull Request を確認し、問題がなければ `main` ブランチにマージする。

### 4. タグとリリースの自動作成
リリース用 PR がマージされると、再度 `tagpr` が実行され、Git タグの作成と GitHub Releases の公開を自動で行う。
